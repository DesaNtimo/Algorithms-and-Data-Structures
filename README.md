# Алгоритмы и Структуры Данных

[![Java](https://img.shields.io/badge/Java-8%2B-blue.svg)](https://www.oracle.com/java/)
[![Education](https://img.shields.io/badge/Course-Algorithms%20%26%20DS-brightgreen.svg)]()

Данный репозиторий представляет собой обширную коллекцию реализаций классических алгоритмов, структур данных и решений олимпиадных/спортивных задач на языке Java. Проект разделен на практические классные работы (семинары) и домашние задания.

## 🗂 Структура репозитория

Весь исходный код и материалы расположены в директории `src/` и логически разделены на две основные ветви:

* **`Practice/`** — Содержит PDF-презентации и материалы с семинаров, а также базовые реализации алгоритмов, разбиравшихся на занятиях.
* **`Homeworks/`** — Содержит условия задач (в формате PDF контестов) и исходный код (Java) с решениями домашних заданий.

## 📚 Темы и Модули

Проект охватывает широкий спектр тем Computer Science, каждая из которых вынесена в отдельную директорию:

### 1. Базовые алгоритмы (`Algorithms`)
* Бинарный и вещественный поиск (Binary Search, Root finding).
* Сортировки (Bubble Sort, Merge Sort, Anti-Qsort).
* Техника двух указателей и базовые математические задачи.

### 2. Базовые структуры данных (`DataStructures`)
* Реализация стеков (включая MinStack), очередей и деков.
* Обратная польская запись (Postfix notation).
* Моделирование структур данных (задачи с вагонами, гоблинами и шариками).

### 3. Деревья (`Trees`)
* Двоичные деревья поиска.
* Балансировка: AVL-деревья (проверка свойств).
* Кучи (Heaps) и пирамидальная сортировка (HeapSort).
* Поиск наименьшего общего предка (Lowest Common Ancestor - LCA).

### 4. Строковые алгоритмы (`Strings`)
* Поиск подстрок, работа с анаграммами.
* Алгоритмы на палиндромах и подпалиндромах.
* Лексикографические сдвиги и сравнения.

### 5. Динамическое программирование (`DynamicProgramming`)
* Наибольшая возрастающая подпоследовательность (LIS).
* Классические задачи ДП: Кузнечик, Платные лестницы, Ход коня.
* Строковое ДП: Расстояние Дамерау-Левенштейна.
* Поиск наибольшего квадрата/подматрицы.

### 6. Деревья отрезков (`SegmentTree` & `SegmentTreeTwo`)
* Базовое дерево отрезков (поиск минимума, K-й единицы, первый элемент не меньше X).
* Дерево отрезков с массовым обновлением (Massive Operations): прибавление на отрезке, присваивание на отрезке, суммы и минимумы.
* Метод сканирующей прямой (Sweep line) и задачи на отрезки/окна.

### 7. Графы (`Graphs` & `GraphsTwo`)
* **Базовые графы:** Обход в глубину (DFS), обход в ширину (BFS), поиск компонент связности, проверка на двудольность, поиск циклов, топологическая сортировка.
* **Продвинутые графы:** Алгоритмы поиска кратчайших путей (Dijkstra и др.), Минимальное остовное дерево (MST).
* Система непересекающихся множеств (Disjoint Set Union - DSU).

### 8. Математика (`Math`)
* Теория чисел: Разложение на простые множители, Гипотеза Гольдбаха.
* Комбинаторика: Биномиальные коэффициенты, факториалы.

### 9. Продвинутое решение задач (`ProblemSolving` & `ProblemSolvingTwo`)
* Жадные алгоритмы, задачи на минимакс, префиксные суммы в матрицах.
* Олимпиадные задачи (Агрессивные коровы, Планировщики, Распил бруса и сжатие строк).

## 🚀 Особенности реализации

* **Производительный Ввод/Вывод:** В большинстве задач директории `Homeworks` используется встроенный статический класс `FastScanner`. Это собственная реализация сканера потока ввода (на базе `BufferedReader` и `StringTokenizer` или чтения байтов), которая необходима для прохождения жестких ограничений по времени (Time Limit) в тестирующих системах.
* **Автономность:** Каждый `.java` файл в домашних заданиях представляет собой независимую и законченную программу с методом `public static void main(String[] args)`, готовую к отправке в проверяющую систему (Яндекс Контест, Codeforces и т.д.).

## ⚙️ Запуск и Использование

Проект можно открыть в любой современной Java IDE:
1. Клонируйте репозиторий на локальную машину.
2. Откройте проект в **IntelliJ IDEA** (проект содержит файл `Algorithms-and-Data-Structures.iml`) или Eclipse.
3. Выберите нужный класс с методом `main`, скомпилируйте и запустите.
4. Данные подаются в стандартный поток ввода `System.in`, результаты выводятся в `System.out`. Тестовые данные для задач можно найти в PDF-файлах контестов в соответствующих папках.
